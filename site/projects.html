Content migration from AuthorIt to Ixiasoft

After an acquisition, management wanted a single tool to maintain all documentation content. The newly-acquired company used AuthorIt and the parent (acquiring) company used Ixiasoft. I analyzed the content sources. Both content sources were in XML but in very different structures. I wrote a Java program to map the structures and perform the transformation.

Link checker to check for broken links

The company's published documentation was hosted on a platform called ZoominDocs. I wrote a REST client that invoked ZoominDocs APIs to crawl the site, get the content of each hosted webpage, validate the links on each page, and generate a user-friendly report of broken links.
