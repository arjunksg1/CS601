What is Kendo UI?

Kendo UI is a JavaScript library by developed by Progress Telerik that enables you to build the UI of a web app. The library provides a wide set of easy-to-use UI components such as grids, text boxes, numeric text boxes, charts, etc. For example, if you want to display a table (a grid in Kendo UI) in a webpage, you need three things:

  • An HTML container for the grid (such as a div tag),
  • a data source, and,
  • the Kendo UI grid widget.

You begin by defining the HTML element with an id:

<div id="grid"></div>

Next, you need a data source that provides the data to be displayed. This could be a REST service or anything that serves data in JSON format. Or, the data can be in the HTML code itself. For example:

var people = [
{ firstName: "John", lastName: "Smith", email: "john.smith@telerik.com" },
{ firstName: "Jane", lastName: "Smith", email: "jane.smith@telerik.com" },
{ firstName: "Josh", lastName: "Davis", email: "josh.davis@telerik.com" },
{ firstName: "Cindy", lastName: "Jones", email: "cindy.jones@telerik.com" }
];

Finally, to initialize a Kendo UI grid widget and bind it to the data source and the HTML element, simply do the following:

$("#grid").kendoGrid({ dataSource: people });

This example uses jQuery but you can also write AngularJS code if you’d prefer, or even just simple JavaScript (the Kendo UI library supports all of these different methods).

To get a feel of how Kendo UI components are used, take a look at the demos: https://demos.telerik.com/kendo-ui/"

What is a CMS?

A CMS, an abbreviation for Content Management System, is a software application that enables you to create websites and manage its different components such as web pages, text blocks, images, videos, etc. A CMS usually provides an easy-to-use interface that lets you:

  • Create web pages
  • Create page layouts and page templates
  • Create, edit, and manage content such as blogs, articles, images, videos, etc
  • Create custom content types (a content type is a sort of a template of properties from which you can create new instances; for example, a ‘News’ content type may have properties such as title, body, author, and date, while a news article, which is an instance of the ‘News’ content type, will have specific values for each of these properties.)

Additionally, a CMS will also have programming interfaces (APIs) that allow for an enormous amount of extensibility. For example, you may want data submitted through your website to be processed in a specific way by the CMS. In such cases, a software developer would use the CMS's programming interfaces to develop code to handle the data. For this reason, CMS's are usually built for a specific type of programming platform--Java, .NET, etc.
